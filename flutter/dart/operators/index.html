Les opérateurs nullable

Les opérateurs nullable sont les opérateurs qui permettent d'avoir la logique suivante

Faire ceci si la variable est null

Opérateur Description
?? Affecte la valeur de l'opérande de droite tant que la valeur de l'opérande de gauche est égale à null
Elle ne s'applique qu'aux variables
??= Affecte tant que la valeur de l'opérande de droite tant que la valeur de l'opérande de gauche est égale à `null`
Elle ne s'applique qu'aux variables
?. Appel une méthode uniquement si l'expression sur laquelle elle est appelé n'est pas nulle. Elle s'utilse sur des méthodes ou fonctions

Utilisation de ??

// Affecte à x, tant que la valeur de newValue est égale à null, alors affecte 1

int x = newValue ?? 1;

Utilisation de ??=

// affecte newValue a x, mais seulement si x faut null, sinon affecte la valeur de x.
var x; 
x ??= newValue;

// La syntaxe littérale est la suivante :

x = x ?? newValue;

Utilisation de ??.

// Si parent est null, n'appelle pas la fonction methode send, et renvoi null.

parent?.send();

L'opérateur de cascade

L'opérateur de cascade vous permet d'effectuer une série d'opérations sur le même objet.

Exemple

class Human {
  final firstName;
  final lastName;
  final age;
  final job;
  final home;
}

class Job {
  final title;
  final company;
}

Person eric = new Person()
    ..firstNAme = 'Eric'
    ..lastName = 'Windmill'
    ..age = 27
    ..job = new Job()
      ..title = 'Engineer'
      ..company = 'InternetAppCompany'
    ..home = 'Louisville, KY';

Les opérateurs de type

Les opérateurs de type vous permettent de vérifier les types et d'attribuer des types

Opérateur Description
is Vérifie si le type est
is! Vérifie si le type n'est pas
as Convertir ou affecte un type

Exemple (is et is!)

int x = 9;
if (x is int) {
  // code here will run
}

if (x is! int) {
  // code here won't run
}

Exemple de as

class Employee {
  // ... 
}

class Engineer {
  //...
}

class Designer {
  //...
}

void newHire() {
  // Cette variable aura toutes les propriétés de la classe Employee
  // Mais son type sera de la classe Engineer.
  var emp345 = new Employee() as Engineer;

  // Cela ne fonctionnera pas tant que emp345 n'est pas Designer
  (emp345 as Designer).team = 'Web Client';
}
